## Mandatory Rules

> Frozen on 2026-05-08 from `~/brain/wiki/dev/identity.md` and `~/brain/wiki/dev/workflows/ai-collaboration.md`. If the wiki updates, this section does not auto-update — re-pull manually.

### TypeScript

- Strict mode always — `strict: true` in `tsconfig.json`.
- Never use `any` — use `unknown` when the type is truly unknown.
- Interface prefix: `I` — `IProject`, `IExperience`.
- Interfaces for object shapes, types for unions/intersections.
- Types live in dedicated `*.types.ts` files or `schema.ts` (with Zod) — never inline in `.tsx` components.

### Naming

| Context                         | Convention               | Example                    |
| ------------------------------- | ------------------------ | -------------------------- |
| Variables, functions, methods   | camelCase                | `getProjects`, `isVisible` |
| Classes, components, interfaces | PascalCase               | `ProjectCard`, `IProject`  |
| Interface prefix                | `I` + PascalCase         | `IProject`                 |
| Constants                       | SCREAMING_SNAKE_CASE     | `MAX_PROJECTS`             |
| Booleans                        | is/has/can/should prefix | `isLoading`, `hasError`    |
| Event handlers                  | handle prefix            | `handleSubmit`             |
| Component files                 | PascalCase               | `ProjectCard.tsx`          |
| Feature folders                 | kebab-case               | `project-showcase/`        |

### React

- Functional components only — no class components.
- **Default export for components**, named exports for everything else.
- One component per file — file name matches component name (PascalCase).
- Custom hooks for shared logic (`use*`).
- Props destructuring in the function signature.
- Barrel files (`index.ts`) per feature folder — explicit named exports, not `export *`.

### Functions & Files

- Single responsibility — one function, one job.
- Max ~50 lines per function (SOLID drives this, not an arbitrary limit).
- Max ~500 lines per file.
- Pure where possible; early returns / guard clauses at the top, happy path at the bottom.

### Async

- Always `async/await`, never `.then()` chains.
- Never `await` inside loops — use `Promise.all()` / `Promise.allSettled()`.
- Always wrap in `try/catch` or rely on framework error boundaries — never swallow errors.

### Imports

- Path aliases always — `@/` for imports beyond one level.
- Barrel imports — import from feature index, not deep internal files.
- Order: external → aliases → relative (auto-sorted by ESLint/Prettier — see `eslint-prettier.md`).

### Variables

- `const` by default; `let` only when reassignment is necessary.
- Never `var`.
- Destructure immediately.
- Declare close to first use.

### Comments

- WHY, not WHAT.
- No trivial comments (e.g. `// increment counter` on `count++`).
- TODO format: `// TODO(hadi): description — YYYY-MM-DD`.

### Package Management

- Always run `pnpm add <package>` directly — never pin a specific version manually, never hand-edit `package.json` then run install. That bypasses the resolver.
- If a specific version is genuinely required, document the reason in `CONTEXT.md` or in a comment near the lockfile.

### What to Avoid

`any`, nested ternaries, magic numbers, commented-out code, `console.log` in commits, mutating function arguments, files over 500 lines, layer-based flat folders, god components, skipping input validation, manually pinned versions.

### What AI Should NOT Decide

> From `~/brain/wiki/dev/workflows/ai-collaboration.md`.

- **Architecture** on complex features — AI proposes, human decides.
- **Technology choices** — stack is picked; AI works inside it.
- **Business logic interpretation** — human owns the domain.
- **Security-critical code** — auth, permissions, data access reviewed by human before shipping.

### Testing — Relaxed for This Project

- TDD and 80%+ coverage are **NOT enforced** for this portfolio (per project owner — frontend-only marketing site, low business risk per page).
- Still recommended: Playwright E2E for any submission flow added later (e.g. contact form).
- Bug-fix tests still encouraged: write a failing test that reproduces the bug, then fix.
- Default wiki testing rule (`~/brain/wiki/dev/patterns/testing.md`) is a reference here, not a gate.

### Local Workflow Rules (per project owner)

- **Never run `pnpm dev` / `pnpm start`.** Hadi keeps the dev server running in his own terminal at all times. If you need the live URL, assume it is already up at `http://localhost:3000`.
- **Browser / Chrome MCP validation is user-gated.** Do not auto-screenshot, auto-navigate, or auto-validate UI in the browser. Wait until Hadi explicitly says something like "run the MCP for this" or "check it in the browser" before invoking any `mcp__claude-in-chrome__*` or `mcp__chrome-devtools__*` tool. Until then, rely on `pnpm typecheck` + `pnpm build` + reading the source.
- **Scope lint and format to changed files only.** When a change touches a couple of files, run `pnpm exec eslint <file1> <file2> ...` and `pnpm exec prettier --write <file1> <file2> ...` against those exact paths. Do NOT run `pnpm lint` or `pnpm format` (which walk the entire project) on small diffs — wasteful and noisy. Project-wide runs are reserved for explicit cleanup passes Hadi asks for.

## Animation Guidelines (project-specific — not yet in wiki)

For Awwwards/FWA-grade portfolio animations:

- Use the **`awwwards-animations`** skill in `~/.claude/skills/` for ScrollTrigger, Lenis smooth scroll, magnetic effects, reveal animations, custom cursors, page transitions, parallax, pin sections, stagger, kinetic typography.
- Use the **`gsap`** skill for HUD-style and complex timeline work.
- Default animation stack: **GSAP + `useGSAP` hook + ScrollTrigger** for scroll-driven; **Lenis** for smooth scroll; **Motion / Framer Motion** for simple component transitions.
- Always wrap GSAP in the **`useGSAP` hook** (from `@gsap/react`) for automatic cleanup — never raw `useEffect`.
- Always respect **`prefers-reduced-motion`** — every animated component must have a reduced-motion fallback.
- Animate **`transform` + `opacity` only** on scroll-tied animations. Aim for 60fps. Avoid layout-thrashing properties.
- Do not animate hero/headline content with anything that delays Largest Contentful Paint — use server-rendered text, animate after paint.

## Performance & Quality Gates

The portfolio's job is to generate interviews and freelance leads. First impression matters.

- Lighthouse target: **90+** on Performance / Accessibility / Best Practices / SEO.
- Use Next.js `Image` component for all images (never raw `<img>`).
- Use Next.js `Link` for all internal links (never raw `<a>`).
- Animations must not block LCP or interaction.

## Design Tokens

Tokens live in **`app/globals.css`** inside the `@theme inline { ... }` block (Tailwind 4 convention). They mirror `DESIGN.md`'s frontmatter.

- Colors generate utilities: `bg-paper-cream`, `text-ink-near-black`, `bg-press-red`, `border-warm-tan-border`, `text-code-lime`, etc.
- Fonts generate utilities: `font-display` (Newsreader), `font-body` / `font-sans` (Inter), `font-mono` (JetBrains Mono).
- Custom letter-spacing utilities: `tracking-kicker` (0.18em), `tracking-label` (0.06em), `tracking-display-tight`, `tracking-headline-tight`.
- Motion tokens are CSS variables: `var(--ease-out-expo)`, `var(--ease-out-quart)`, `var(--duration-fast)`, etc.

When adding a new token, update both `DESIGN.md` (frontmatter + prose), `DESIGN.json` (sidecar), and `app/globals.css` together. Token drift between the three is a slop tell.

## Animation Setup (already wired)

- **GSAP**: import from `@/lib/gsap` (NOT directly from `gsap`). The lib file registers ScrollTrigger client-side. Wrap every GSAP timeline in the `useGSAP` hook from `@gsap/react`:
  ```ts
  import { useGSAP } from "@gsap/react";
  import { gsap } from "@/lib/gsap";
  ```
- **Lenis**: import `lenisDefaultOptions` and `prefersReducedMotion` from `@/lib/lenis`. The Lenis Provider component itself is not yet built — create it as `components/providers/LenisProvider.tsx` when starting UI work, gated on `!prefersReducedMotion()`.
